## 概述

## 一、完整故事分析工作流

### 场景：深度理解一个故事

```
用户输入：请帮我深度分析这个故事 [提供故事文本]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案 1：使用组合 Skills
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: 使用 story-outliner
AI：让我先用 story-outliner 生成故事大纲...
     [生成300-500字的故事大纲]

Step 2: 使用 story-type-analyzer
AI：现在用 story-type-analyzer 分析题材类型...
     [分析题材类型、创意元素]

Step 3: 使用 plot-keypoints
AI：接下来用 plot-keypoints 梳理大情节点...
     [按阶段列出主要情节点]

Step 4: 使用 character-profile
AI：然后用 character-profile 生成人物小传...
     [为每个主要人物生成小传]

Step 5: 使用 character-relationships
AI：再用 character-relationships 分析人物关系...
     [分析人物关系网络]

Step 6: 使用 mind-map-generator
AI：最后生成思维导图可视化...
     [提供思维导图链接]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案 2：一键完成（推荐）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

使用：story-five-elements

AI：我使用 story-five-elements 为您进行全面分析...
     [自动完成以上所有步骤，生成综合报告]
```

```
用户输入：这个小说值得改编成短剧吗？[提供小说信息]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
IP 评估决策流程
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: IP 初步评估
使用：ip-evaluator

AI：我先使用 ip-evaluator 对这个 IP 进行初步评估...
     [分析市场潜力、创新属性、内容亮点]

评估结果：总评分 8.2分
判断：有潜力，进入下一轮评估

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 2: 故事大纲评估
使用：story-outline-evaluator (需要先生成大纲)
前置：story-outliner

AI：让我先生成故事大纲，然后进行详细评估...
     使用 story-outliner 生成大纲...
     使用 story-outline-evaluator 进行评估...

评估结果：总评分 7.8分
判断：合格但需要优化，进入下一轮评估

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 3: 短剧改编潜力评估
使用：drama-evaluator

AI：现在用 drama-evaluator 评估短剧改编潜力...
     [分析核心爽点、故事类型、人物设定等]

评估结果：总评分 8.1分
判断：有较强的改编潜力

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 4: 综合建议

AI：【综合评估建议】

基于三轮评估结果：
- IP 初步评估：8.2分（有潜力）
- 大纲评估：7.8分（需要优化）
- 短剧潜力：8.1分（较强潜力）

总体判断：建议推进，但需要进行优化

优化建议：
1. 增强核心爽点的极致性
2. 优化人物设定的标签化
3. 加强情节的戏剧张力
```

使用：series-analyzer

自动执行以下分析：

   - 导演、演员、评分等基础信息

【已播剧集分析报告】
一、剧集信息
二、五元素分析
三、拉片分析
四、联网搜索信息

## 五、长文本深度分析工作流

### 场景：分析长篇小说

```
用户输入：深度分析这部长篇小说 [提供超长文本]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
长文本处理流程
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

使用：drama-workflow

AI：检测到文本较长，我使用 drama-workflow 进行处理...

自动编排执行：

1. 文本预处理
   - text-truncator: 智能截断到合适长度
   - text-splitter: 分割成可处理的文本块

2. 并行分析
   - 对每个文本块并行执行 drama-analyzer
   - 分析情节点和戏剧功能

3. 结果整合
   - result-integrator: 整合所有分析结果
   - 去除重复内容
   - 按戏剧功能分类
   - 按故事顺序排序

4. 报告生成
   - 生成综合分析报告
   - 提供整体戏剧结构评价

输出进度：
[25%] 文本预处理完成...
[50%] 并行分析进行中...
[75%] 结果整合中...
[100%] 分析完成！
```

## 八、快速决策工作流

### 场景：快速判断是否继续推进

```
用户输入：快速判断这个故事值不值得做

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
快速评估流程
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

使用：drama-evaluator (短剧评估专家)

AI：我使用 drama-evaluator 为您快速评估...

核心评估维度：
1. 核心爽点（评分）
2. 故事类型（评分）
3. 人物设定（评分）
4. 人物关系（评分）
5. 情节桥段（评分）

快速判断：
- 总评分 >= 8.5：优秀，直接推荐
- 总评分 8.0-8.4：有潜力，需要优化
- 总评分 7.5-7.9：一般，需要大幅修改
- 总评分 < 7.5：较差，不建议推进

输出：
【快速评估结果】
总评分：X.X分
判断：[推荐/优化后推荐/不建议]
核心建议：[1-2条关键建议]
```
