### 1.2 Description 编写规则

**格式模板**:
```
[动作动词] + [对象] + [核心功能描述]。适用于 [场景1]、[场景2]、[场景3]
```

**好的 Description 示例**:
- ✅ `分析故事文本，提炼主要情节点并分析戏剧功能。适用于分析小说、剧本大纲、故事梗概等文本，识别关键转折点和情感节点`
- ✅ `智能截断文本，保持内容的完整性和语义连贯性。适用于长文本预处理、确保文本不超过指定长度限制`
- ✅ `梳理IP网络信息并进行多维度评估打分。适用于评估小说、剧本等IP的改编价值、分析市场潜力和创新属性`

**差的 Description 示例**:
- ❌ `这个技能是用来分析剧本的` (缺少使用场景)
- ❌ `分析故事文本` (过于简单，缺少场景)
- ❌ `这是一个非常强大的工具，可以帮助用户进行各种复杂的分析任务` (过于宽泛，缺少具体功能)

### 1.3 命令式语言规范

| 错误（被动/描述式） | 正确（命令式） |
|-------------------|--------------|
| 应该先分析... | 先分析... |
| 需要进行提取... | 提取... |
| 建议用户... | 让用户... |
| 可以考虑... | 考虑... |
| 你可以... | ... |
| 应当... | ... |

**标准动词列表**:
- `分析` 而非 `进行分析`
- `提取` 而非 `进行提取`
- `生成` 而非 `进行生成`
- `评估` 而非 `进行评估`
- `创作` 而非 `进行创作`
- `优化` 而非 `进行优化`

# 剧本分析专家

## 功能

分析故事文本，提炼主要情节点并分析每个情节点的戏剧功能。

## 使用场景

- 分析小说或故事文本的核心情节结构
- 识别故事中的关键转折点和情感节点
- 评估每个情节点的戏剧作用和推动力
- 为剧本改编提供情节分析基础

## 核心步骤

1. **深度阅读**: 充分阅读和理解故事文本内容
2. **情节点提炼**: 根据情节点定义，总结故事中的主要情节点
3. **戏剧功能分析**: 分析每个情节点的戏剧作用
4. **结构化输出**: 按照指定格式输出分析结果

## 输入要求

- 完整的故事文本（小说、剧本大纲、故事梗概等）
- 文本长度建议：500字以上
- 支持多种文本格式：纯文本、Markdown等

## 输出格式

```
【情节点】：<单个情节点描述>
【戏剧功能】：<该情节点的戏剧功能分析>

【情节点】：<单个情节点描述>
【戏剧功能】：<该情节点的戏剧功能分析>
...
```

## 约束条件

- 每个情节点的表述不超过100字
- 至少提炼5个情节点
- 严格按照故事文本原文意思总结，不自行创作改编
- 不使用阿拉伯数字为情节点标号
- 避免幻觉，不要将提示词内的词句带进生成的答案中

李铭前往指定地点，发现是一个废弃的仓库。在仓库中，他找到了一本日记，日记的主人正是他父亲生前的同事。日记中记录了一个惊人的秘密：父亲的死并非意外，而是被人谋杀。

## 六、常见问题

### Q1: allowed-tools 字段何时需要？

**A**: 当 skill 需要调用特定工具时添加。例如：
- 需要文件读写：`allowed-tools: [Read, Write]`
- 需要网络搜索：`allowed-tools: [Search]`
- 仅纯文本处理：可省略

### Q2: model 字段何时需要？

**A**: 当 skill 需要特定模型能力时添加。例如：
- 复杂创作任务：`model: sonnet` 或 `model: opus`
- 简单分析任务：可省略（使用默认模型）
- 快速处理任务：`model: haiku`

### Q3: 如何决定是否创建 references/ 目录？

**A**: 
- 主 SKILL.md 超过 4000 词：创建 references/
- 需要多个详细示例：创建 `references/examples.md`
- 有复杂的使用指南：创建 `references/guide.md`
- 简单 skill（<3000 词）：可省略 references/
